# AutoSettlement 사용자 테스트 가이드

이 문서는 개발자가 아니라 실제 사용자가 화면을 직접 눌러보며 확인할 항목만 정리한 테스트 가이드입니다.

## 테스트 전 준비

1. 담당자가 전달한 테스트 URL을 준비합니다.
   - 테스트 URL: `https://autosettlement.vercel.app/`
   - `http://127.0.0.1:5173/` 주소는 개발자 PC에서만 열리는 로컬 주소이므로 실사용자 테스트용으로 전달하면 안 됩니다.
2. 실사용자 PC의 브라우저에서 전달받은 테스트 URL을 엽니다.
3. 첫 화면 제목이 아래처럼 보이는지 확인합니다.
   - `정산 작업 목록 / 작업 시작`
4. 첫 화면 버튼이 아래 2개로 보이는지 확인합니다.
   - `라온이앤엠 정산`
   - `에스알이앤엠 정산`

## 테스트 원칙

- 이상한 문구가 있으면 캡처해서 공유합니다.
- 파일 업로드가 막혀야 하는데 가능하거나, 가능해야 하는데 막히면 기록합니다.
- 다운로드가 가능해야 할 것 같은데 안 되면 현재 화면의 `상태`, `이슈`, `검수`, `출력` 영역을 같이 캡처합니다.
- `배치`, `실파일 업로드`, `정산 작업 열기`, `새 정산 작업 시작` 같은 이전 용어가 보이면 버그로 기록합니다.

---

## 1. 첫 화면 문구 테스트

### 확인 절차

1. 브라우저에서 전달받은 테스트 URL에 접속
2. 첫 화면 확인

### 기대 결과

다음 문구가 보여야 합니다.

- `정산 작업 목록 / 작업 시작`
- `라온이앤엠 정산`
- `에스알이앤엠 정산`

다음 문구는 보이면 안 됩니다.

- `배치`
- `라온이앤엠 새 정산 작업 시작`
- `에스알이앤엠 새 정산 작업 시작`
- `이 정산 작업 열기`
- `정산 작업 열기`

---

## 2. 라온이앤엠 정산 진입 테스트

### 확인 절차

1. 첫 화면에서 `라온이앤엠 정산` 클릭
2. 업로드 화면 확인

### 기대 결과

- 상단에 `라온이앤엠 업로드 영역`이 보여야 합니다.
- `회사 모드: 라온이앤엠`이 보여야 합니다.
- 플랫폼 카드들이 보여야 합니다.
- 원스토어 카드에는 `원스토어 (공유)`가 보여야 합니다.
- 업로드 버튼 문구는 `파일 업로드`여야 합니다.

다음 문구는 보이면 안 됩니다.

- `실파일 업로드`

---

## 3. 에스알이앤엠 정산 진입 테스트

### 확인 절차

1. 첫 화면으로 돌아갑니다.
2. `에스알이앤엠 정산` 클릭
3. 업로드 화면 확인

### 기대 결과

- 상단에 `에스알이앤엠 업로드 영역`이 보여야 합니다.
- `회사 모드: 에스알이앤엠`이 보여야 합니다.
- 플랫폼 카드들이 보여야 합니다.
- 원스토어 카드에는 `원스토어 (공유)`가 보여야 합니다.
- 업로드 버튼 문구는 `파일 업로드`여야 합니다.

다음 문구는 보이면 안 됩니다.

- `실파일 업로드`

---

## 4. 파일 업로드 문구 테스트

### 확인 절차

1. 라온이앤엠 또는 에스알이앤엠 정산 화면에 진입합니다.
2. 업로드 가능한 플랫폼 카드의 버튼 문구를 확인합니다.

### 기대 결과

업로드 전 버튼은 아래 문구여야 합니다.

- `파일 업로드`

업로드 연결이 아직 없는 위치는 아래 문구여야 합니다.

- `파일 업로드 연결 예정`

다음 문구는 보이면 안 됩니다.

- `실파일 업로드`
- `실업로드 연결 예정`

---

## 5. 잘못 올린 파일 교체 테스트

### 확인 절차

1. 업로드 가능한 플랫폼에서 파일을 하나 업로드합니다.
2. 업로드 후 해당 카드의 버튼 문구를 확인합니다.
3. 같은 버튼을 다시 눌러 다른 파일을 선택해 봅니다.

### 기대 결과

업로드 후 버튼은 아래처럼 바뀌어야 합니다.

- `파일 교체`

아래 안내 문구가 보여야 합니다.

- `잘못 올린 파일은 다시 업로드하면 교체됩니다.`

새 파일을 다시 올리면 기존 파일명/결과가 새 파일 기준으로 바뀌어야 합니다.

---

## 6. 원스토어 공통 업로드 테스트

원스토어는 라온이앤엠과 에스알이앤엠이 공유하는 공통 업로드입니다.

### 확인 절차

1. `라온이앤엠 정산`으로 진입합니다.
2. `원스토어 (공유)` 카드에서 파일을 업로드합니다.
3. 첫 화면 또는 정산 작업 목록으로 돌아갑니다.
4. `에스알이앤엠 정산`으로 진입합니다.
5. `원스토어 (공유)` 카드를 확인합니다.

### 기대 결과

에스알이앤엠 화면의 원스토어 업로드 버튼/input은 비활성화되어야 합니다.

아래 안내 문구가 보여야 합니다.

- `라온이앤엠에서 공통 업로드 완료 · 수정은 라온이앤엠 정산에서 진행`

즉, 라온에서 원스토어를 이미 올렸으면 에스알에서는 중복 업로드를 못 해야 합니다.

---

## 7. 원스토어 공통 업로드 교체 테스트

### 확인 절차

1. 원스토어 파일을 처음 업로드했던 회사 화면으로 돌아갑니다.
   - 예: 라온이앤엠에서 업로드했다면 `라온이앤엠 정산`
2. `원스토어 (공유)` 카드를 확인합니다.

### 기대 결과

- 원스토어 버튼은 비활성화되면 안 됩니다.
- 버튼 문구는 `파일 교체`여야 합니다.
- 아래 문구가 보여야 합니다.
  - `잘못 올린 파일은 다시 업로드하면 교체됩니다.`

즉, 원스토어 수정은 처음 업로드한 회사 정산 화면에서 해야 합니다.

---

## 8. PASS 테스트

### 확인 절차

1. 필수 파일이 없는 플랫폼 또는 슬롯을 찾습니다.
2. `파일 없음으로 PASS 처리` 버튼이 보이는지 확인합니다.
3. 실제로 해당 파일이 없는 상황이라면 PASS 버튼을 누릅니다.

### 기대 결과

- 해당 항목이 `업로드 없음 PASS` 상태로 표시되어야 합니다.
- 해당 파일 누락 이슈가 사라져야 합니다.
- 다른 플랫폼/다른 슬롯의 이슈는 사라지면 안 됩니다.
- 다른 blocker가 없으면 다운로드 가능 상태로 넘어가야 합니다.

---

## 9. 검수/출력 기본 흐름 테스트

### 확인 절차

1. 업로드 또는 PASS 처리를 진행합니다.
2. `파싱/상태 확인` 영역에서 이슈가 남아 있는지 확인합니다.
3. `공통 정산 검수` 영역에서 검수 대기/보류/확정 상태를 확인합니다.
4. `회사별 출력` 영역에서 다운로드 가능 여부를 확인합니다.

### 기대 결과

다운로드가 가능하려면 일반적으로 아래 조건이 충족되어야 합니다.

- 필수 업로드가 완료 또는 PASS 처리됨
- unresolved issue가 없음
- 검수 row가 확정됨
- 출력 validation이 통과함

다운로드가 막힌다면 화면에서 막힌 이유가 보여야 합니다.

---

## 10. 사용자 테스트 결과 기록 양식

테스트 중 문제가 있으면 아래 형식으로 기록합니다.

```text
[문제 위치]
예: 라온이앤엠 정산 > 원스토어 카드

[무엇을 했는지]
예: 원스토어 파일을 라온에서 업로드한 뒤 에스알 화면으로 이동

[기대 결과]
예: 에스알 원스토어 업로드 버튼이 비활성화되어야 함

[실제 결과]
예: 에스알에서도 파일 업로드 버튼이 눌림

[첨부]
스크린샷 또는 파일명
```

## 테스트 완료 기준

사용자 테스트 완료 기준은 아래입니다.

- 첫 화면 용어가 자연스럽다.
- `배치`, `실파일 업로드` 같은 어색한 이전 용어가 보이지 않는다.
- 라온/에스알 정산 진입이 헷갈리지 않는다.
- 원스토어 공통 업로드가 한쪽에서만 수정 가능하다.
- 잘못 올린 파일을 `파일 교체`로 수정할 수 있다.
- 의도적으로 없는 파일은 PASS 처리할 수 있다.
- 출력이 막힐 때 이유가 화면에서 이해된다.
